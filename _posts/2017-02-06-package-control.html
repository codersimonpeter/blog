今天像往常一样打开package control，像往常一样输入install ，但是却发现找不到install package的选项了。
我的第一反应就是，难道sublime被损坏了？于是我卸载了sublime 3 ，并在官网重新下载了sublime text 3 的最新版本。官网地址然后重新安装package control。
PS：安装代码会不停的更新，所以有时候在网上找的安装code不一定能用。在官网去找就好了Package Control官网地址。
但是，在console输入安装code之后 ，程序假死，然后过了一段时间弹出错误码:275309

然后就没有然后了。。。
这个时候你再打开package control面板的时候，依然是没有install package的选项的。翻了一会儿Google之后，在stackoverflow上面发现这样一篇帖子

When I upgraded to 3124 (OSX El Capitan), I noticed that I didn't have Package Control installed anymore (the command palette didn't have Package Control items). The release notes for 3124 mentioned that SublimeText baked in a way to install Package Control, so I looked for that option in the command palette and couldn't find it. It seems that, for some reason, Package Control was added to my ignored_packages array in my preferences. I'm not really sure why, but removing package control from the ignore packaged list brought everything back to normal for me.
即sublime升级到build 3124的时候（当前build3126依旧没有修复）package control被移到了ignored_packages里面。也就是说，其实package control并没有出现问题，只是被设置忽略掉了（因为被莫名移动到ignored_packages）。
因此，从ignored_packages 清单移除package control 就可以解决了（或者删掉整个ignored—packages）。

解决办法：
CMD+,， 或者Sublime Text -> Preferences -> Setting进入设置（Windows或汉化版下请自行对照），把package control那行删掉或者注释掉，保存，重启sublime text就可以了。
顺带一提，在sublime官网里面发现了3124的更新说明，里面提到
Added a menu item and command palette entry to install Package Control
然而我并没有找到该菜单选项。
